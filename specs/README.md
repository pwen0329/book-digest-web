# Feature Specifications

Write a spec here **before** building any new feature.

## Workflow

1. Copy `TEMPLATE.md` to `features/your-feature-name.md`
2. Fill out the spec
3. Start coding
4. Update the spec if reality diverges

## Directory

```
specs/
├── TEMPLATE.md          # Copy this for new features
└── features/            # Your feature specs go here
    └── example-feature.md
```

That's it. Keep specs close to the code by linking to implementation files.
